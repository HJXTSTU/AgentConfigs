# C#代码规范

## 使用场景
当需要对修改的cs代码文件评审时使用。

## 代码风格说明
1. 类名首字母大写
2. 成员变量命名格式: m_Xxxxx
3. 成员函数首字母大写
4. 左花括号({)和右花括号(})需要独占一行。内部没有内容时，不得在同一行。
5. 静态成员变量命名格式：s_Xxxxx
6. Property的首字母大写，且没有m_和s_前缀，参考如下示例:
```
public int Count
{
    get;set;
}
```

### 中风险

### 低风险
1. 代码编写不符合规范
